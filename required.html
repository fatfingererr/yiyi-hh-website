商業模式畫布圖 Business Model Canvas

1. 了解 BMC 各個區塊與涵義
2. 能講解寫 BMC 的基本原則（列點，可驗證）
3. 能指出初期的 BMC 和最終的 BMC 可能差異之處

顧客開發 Customer Development

1. 能說明如何進行 BMC 的驗證
2. 能解釋 CD 的核心原則（走出辦公室）
3. 能指出 CD 的前兩個階段：顧客探索 vs. 顧客開發的差異

顧客訪談 Customer Interview

1. 能設計一個可測試的商業實驗
2. 知道訪談前需要做的哪些準備
3. 能從訪談流程中和結束後，擷取有用的建議

建立代言人檔案 Customer Profiling

1. 明白如何拆解目標客群的一天（B=MAT）
2. 能進行九宮格情境分析
3. 能繪製客戶的利益關係圖（決策者，購買者，使用者）

最小可行產品設計 MVP Design

1. 知道最小可行產品與一般產品的差別
2. 熟練於沉澱出 MVP 的功能 (MoSCoW)
3. 能快速地在 MVP 的功能上進行產品迭代

掌握軸轉時機 Time to Pivot

1. 能判別 MVP 在什麼情況下應該進行 Pivot
2. 能指出常見的 Pivot 方式與結果
3. 能發現什麼是正確的 Pivot，還是錯誤的 Pivot

資料、檔案與權限 Data and File Storage & Premission

1. 熟悉使用 Google Drive (包含 Google Docs, Google Slides)
2. 熟悉使用日程與會議管理 Google Calendar
3. 會以 AgileCRM 進行簡易的客戶資料管理

協作工具 Collaboration

1. 掌握如 GitHub, ZenHub 等軟體開發協作技巧
2. 能有效透過 Slack, Trello 進行團隊交流
3. 熟練於透過 Zoom 的各種功能進行線上會議

知識共享 Knowledge Sharing & Reuse

1. 掌握 Markdown 語法與常見的 README 寫法
2. 了解常見的軟體專案的 Wiki 基本架構
3. 能設計團隊內部知識對外分享的方式（Blog Post, Conf）

敏捷心態 The Agile Manifesto

1. 熟練掌握敏捷開發的核心原則
2. 能分辨敏捷開發流程的各個角色
3. 明白什麼是看板方法（Kanban）

認識 Scrum Understanding Scrum

1. 知道什麼是迭代（An Iteration）
2. 能進行衝刺（Sprint）和衝刺會議
3. 能說明單元測試（Unit Test）在衝刺中扮演的角色

交付管理 Release Management

1. 知道如何設定要求的基本大小（Sizing）
2. 能觀察如何設定團隊前進的速度（Velocity）
3. 能講清楚每一次的交付要追蹤的重點

文件撰寫技巧 Documentation

1. 知道如何撰寫使用者故事（User Stories）
2. 能明白非功能性需求應該如何被考量
3. 能設定還需要紀錄的最少必要文件

程式碼設計範式 Programming Paradgm

1. 知道物件導向程式設計（OOP）與 SOLID 原則
2. 知道函數式程式設計（FP）與副作用（Side Effects）
3. 知道並行程式設計 & 非同步程式設計

寫出可測試的程式 Testability Patterns

1. 熟練使用單元測試（Unit Test）與重構（Refactoring）
2. 能講解測試路徑與使用者故事的關係
3. 能訂定撰寫測試的基本原則

程式碼審查 Code Review

1. 能說明快速審查（short reivew）與版本管理的關係
2. 能設定程式碼審查判斷的標準（維護性，錯誤處理，註解與文件）
3. 能解決團隊在程式碼審查中可能發生的衝突

API 設計與模組化 API Design and modularity

1. 知道 SOA 架構設計與技術故事（Tech Story）
2. 知道系統模組化會使用到的設計模式（Design Patterns）
3.能思考 API 設計與版本管理的關係（審查，相容，可擴展性）

持續整合 Continuous Integration

1. 了解什麼是 Continuous Inspection 和 Continuous Testing
2. 能清楚思考 CI 與版本控制的關係
3. 知道如何建立 CI 的 Pipeline

持續交付 Continuous Delivery

1. 明白什麼是 Continuous Deployment 和基礎設施管理
2. 知道如何進行環境管理與藍綠佈署
3. 知道如何建立 CD 的 Pipeline

建立 DevOps 防護機制 Securing DevOps Pipeline

1. 知道如何保護網路應用（Web Applications）的安全
2. 知道如何建立 DevOps pipeline 的安全層（Security layer）
3. 知道如何建立監控的日誌紀錄流程
